Android Gradle projects use a different directory structure than ADT
Eclipse projects. Here's how the projects were restructured:

* AndroidManifest.xml => app\src\main\AndroidManifest.xml
* assets\ => app\src\main\assets
* libs\armeabi\libWlt.so => app\src\main\jniLibs\armeabi\libWlt.so
* libs\json.jar => app\libs\json.jar
* res\ => app\src\main\res\
* src\ => app\src\main\java\
* src\resources\hibernate\hibernate.properties => app\src\main\resources\resources\hibernate\hibernate.properties
* src\resources\locale\Language_en_US.properties => app\src\main\resources\resources\locale\Language_en_US.properties
* src\resources\locale\Language_zh_CN.properties => app\src\main\resources\resources\locale\Language_zh_CN.properties
* src\resources\locale\platform_zh_CN.properties => app\src\main\resources\resources\locale\platform_zh_CN.properties
* src\resources\log4j\log4j.properties => app\src\main\resources\resources\log4j\log4j.properties
* src\resources\spring\applicationContext-persistence.xml => app\src\main\resources\resources\spring\applicationContext-persistence.xml
* src\resources\spring\applicationContext.xml => app\src\main\resources\resources\spring\applicationContext.xml
* src\resources\struts\struts.xml => app\src\main\resources\resources\struts\struts.xml

Next Steps:
-----------
You can now build the project. The Gradle project needs network
connectivity to download dependencies.

Bugs:
-----
If for some reason your project does not build, and you determine that
it is due to a bug or limitation of the Eclipse to Gradle importer,
please file a bug at http://b.android.com with category
Component-Tools.
